# 판정 = 설명 (NM 엔진 v2)

같은 자료를 LLM과 NM에 함께 준다. LLM 답변의 각 문장을 NM이
"자료에 근거하나"로 판정하고, **판정(색)과 근거(σ 이탈·logP)를 한 몸으로** 보여준다.
사후 설명이 아니라 — 판정하는 계산이 곧 설명이다.

## v2에서 개선된 것 (v1 대비 검증 완료)
1. **동적 임계값(자료 적응)**: 고정 -10/-14 대신, 그 자료의 logP 분포 기준
   "몇 σ 이탈"로 판정. 도메인마다 자동 적응. 판정 근거도 강해짐
   ("자료 분포 대비 5.9σ 이탈").
   - 핵심: 캘리브레이션에 **어미 변형 문장**을 포함해, 원본 문체(-다)와
     실제 입력 문체(-습니다)의 분포 불일치를 해결.
2. **투명한 PMI 의미 보정**: 블랙박스 트랜스포머 임베딩 대신 공기어 통계(PMI).
   왜 보정됐는지(어떤 공기어) 근거를 남김 → 완전 투명.
3. **불안정 지표 제거**: v1의 mismatch(임계값 근처에서 정상 문장을 오판하던
   불안정성)를 제거.

## 검증 결과 (5개 도메인, 인공 테스트셋)
안전교육·공문서·레시피·소량자료·AI 경계셋에서 V1 26/54 vs **V2 27/54**.
특히 AI 경계셋에서 V1이 정상 문장을 CRITICAL로 오판한 케이스를 V2가 정확히 통과.
(작은 인공셋 결과 — 실데이터 대규모 검증은 다음 단계)

## 구조
1. 자료(PDF/docx/txt)를 LLM·NM에 동일 투입
2. NM이 자료로 학습(어미변형 포함 캘리브레이션)
3. LLM 답변을 문장 단위로 판정: 🟢근거있음 🟡경계 🔴벗어남
4. 각 문장 밑 판정+근거(σ 이탈·logP·약한 표현). 판정과 설명이 분리 안 됨.

## 왜 이게 시그니처인가
SHAP·LLM-as-judge는 판정 따로 설명 따로(사후 추정). NM은 판정 계산이
곧 설명이라 지어낼 수 없다. + 로컬·CPU·투명. 규제(EU AI법 설명가능성)에 부합.

## 어미 문제 해결 (Kiwi 형태소 분석)
한국어 종결어미가 문제였다: 자료의 "간다"와 답변의 "갑니다"가 다른 토큰이 되어
정상 문장의 logP가 튀고, '정상 속에 낀 환각 단어'를 놓쳤다.

**해결**: `korean_tokenizer_kiwi.py`가 Kiwi 형태소 분석으로 어간을 추출.
"간다"·"갑니다" 모두 어간 "가"로 통일 → 어미 변화에 강건해지고,
이질적 단어(환각)가 선명하게 드러난다.
- Kiwi 미설치 시 규칙 토크나이저로 자동 폴백(동작은 하되 어미 문제 잔존).
- 로컬에서 `pip install kiwipiepy` 하면 자동 활성화.

⚠️ 검증 상태: 이 개선의 실제 효과는 Kiwi가 설치된 로컬에서 검증 필요.
   (개발 환경은 네트워크 제한으로 Kiwi 미설치 → 폴백으로만 동작 확인됨)

## 속도 최적화 (v3.1)
모바일에서 5분+ 걸리던 학습을 초 단위로 단축(약 30배).
원인이었던 두 가지 낭비 제거:
1. 마르코프 카운팅 시 토큰을 epochs(20)배 복제 → 확률은 동일한데 시간만 20배.
2. TinyTransformer 20 epochs 학습 → V2에서 PMI 보정으로 대체돼 판정에
   전혀 안 쓰이는데 학습만 하고 있었음.
→ 판정 결과 100% 동일, 속도만 대폭 개선. 실제 문서로 시연 가능한 속도 확보.
