# nm-demo
# 판정 = 설명 (NM 엔진 시그니처 데모)

**같은 자료를 LLM과 NM에 모두 준다.** LLM이 답하면, 그 답의 각 문장을
NM이 "이 자료에 근거하나"로 판정하고 — **판정(색)과 근거(logP)를 한 몸으로** 보여준다.

## 왜 이게 시그니처인가
일반 XAI(SHAP·LIME·LLM자기설명)는 **판정 따로, 설명 따로**다.
블랙박스가 판정하고 → 나중에 "왜 그랬을까"를 사후 추정한다. 그래서 설명이
진짜 이유가 아닐 수 있다.

이 엔진은 다르다:
> **판정하는 계산이 곧 설명이다. avg_logP가 낮아서 '벗어남'이고,
> 그 logP 값 자체가 근거다. 사후에 덧붙이지 않으니 지어낼 수 없다.**

## 구조
1. 자료(PDF/docx/txt)를 LLM과 NM에 **동일 투입**
2. LLM이 그 자료로 답변 생성
3. NM이 답변을 **문장 단위**로 판정:
   - 🟢 초록 = 자료에 근거함 (logP 높음)
   - 🟡 노랑 = 경계 (일부 이탈)
   - 🔴 빨강 = 자료에서 벗어남 (환각 가능, logP 낮음)
4. 각 문장 밑에 판정+근거 주석. 판정과 설명이 분리되지 않는다.

## 검증됨
자료에 있는 문장 → 초록. LLM이 지어낸 "2019년 국제 협약"(자료에 없음)
→ 빨강(logP -15.3) + 근거 약한 표현 '2019년' 지목. 실제로 환각을 잡아냄.

## 왜 GPT로 검증 안 하나
GPT에게 "네 답 맞아?"라고 물으면 또 그럴듯한 답을 낼 뿐(자기합리화).
NM은 GPT와 독립적으로, 같은 자료 기준으로 판정한다. 그리고 그 판정은
계산이라 지어낼 수 없다. 이게 "GPT를 GPT로 검증하면 안 되는 이유"의 답.

## 특징
- **로컬·CPU**: 자료 학습·판정 모두 로컬. 민감 자료가 밖으로 안 나감.
- **투명**: logP·근거 토큰이 그대로 보임.
- **모체**: 이 데모가 핵심. 도메인(교육/공문서/의료)으로 업그레이드 가능.

## 실행
```bash
pip install -r requirements.txt
streamlit run app.py
```
1. 왼쪽에 자료 넣고 "NM 학습"
2. (선택) OpenAI로 답변 생성 또는 LLM 답변 직접 붙여넣기
3. "문장별 검증" → 색+주석으로 판정=설명 확인

## 구조 파일
- `core/neural_markov_engine.py` : NM 엔진(JM-smoothed, logP 판정)
- `core/verify_core.py` : 문장 분리 + 문장별 판정(판정=설명)
- `core/file_ingest.py` : PDF/docx/txt 추출
- `core/korean_tokenizer.py` : 한국어 토크나이저
- `app.py` : Streamlit UI
